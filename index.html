8. Обработка Формирование плановых затрат на персонал для SAP BW 99

Выполнять проверку данных, сформированных по алгоритмам выгрузки плановых затрат на персонал в смежные учетные системы АСПБ и SAP.

Обработка Формирование плановых затрат на персонал для SAP BW

Обработка предназначена для формирования пакета данных по правилам метода веб-сервиса, который обеспечивает загрузку данных о плановых затратах на персонал из ЗУП в SAP. Создание такой обработки обусловлено необходимостью выполнения предварительной проверки подготовленных данных на стороне ЗУП, до того, как эти данные будут загружаться в SAP.

Важно! Непосредственно выгрузить данные из ЗУП в SAP с помощью обработки Формирование плановых затрат на персонал для SAP BW невозможно. Процедура загрузки плановых затрат на персонал из ЗУП в SAP выполняется на стороне SAP с использование веб-сервиса (веб-сервис ИТ_Интеграция_SAP, метод GetPlanCosts).

По кнопке Печать формируется отчет по данным табличной части обработки. В качестве прототипа использован отчет Форма 2 КО (в части плановых данных). Рекомендуется данные этого отчета сравнить с данными отчета Форма 2 КО, сформированного из раздела меню Бюджетирование - Отчеты. Если данные идентичны, то это означает, что веб-сервисом из ЗУП в SAP будут переданы корректные данные.

Дата изменения иерархии – дата перемещения подразделения к другому родителю. Заполняется текущей датой, изменению не подлежит. Участвует в процедурах консолидации с SAP HR. При создании подразделения дата изменения иерархии устанавливается равной дате ввода подразделения. До записи нового подразделения изменение даты ввода приводит к изменению и даты изменения иерархии. После записи подразделения дата изменения иерархии при изменении даты ввода не меняется.

ИД (SAP HR) – идентификатор для консолидации с SAP HR. Выводится справочно.

Важно. При записи подразделения будет изменен реквизит «Дата изменения иерархии» на текущую дату. Изменение данного реквизита запрещено. Данный реквизит используется при консолидации данных ЗУП и SAP HR.
